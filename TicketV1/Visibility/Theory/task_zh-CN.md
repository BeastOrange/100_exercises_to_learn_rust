## 可见性

当你开始将代码分解成多个模块时，你需要开始考虑**可见性**。  
可见性决定了代码的哪些区域（包括其他人的代码）可以访问某个实体，例如结构体 (struct)、函数、字段等。

## 默认是私有的

默认情况下，Rust 中的所有内容都是**私有的**。  
一个私有的实体只能在以下情况下被访问：

1. 在定义它的同一模块内；或  
2. 通过它的某个子模块  

在之前的练习中，我们广泛使用了这一点：

- `create_todo_ticket` 之所以能工作（在你添加了 `use` 声明之后），是因为 `helpers` 是 crate 根模块的子模块，而 `Ticket` 是在 crate 根模块中定义的。  
  因此，即使 `Ticket` 是私有的，`create_todo_ticket` 也可以毫无问题地访问它。  
- 我们的所有单元测试都定义在它们所测试代码的子模块中，因此它们可以没有限制地访问所有内容。

## 可见性修饰符

你可以使用**可见性修饰符**修改实体的默认可见性。  
一些常见的可见性修饰符是：

- `pub`：将实体设为**公共的**，即在定义它的模块外部也可以访问，甚至可能是在其他 crate 中访问。  
- `pub(crate)`：将实体设为在同一个**crate**内公共，但在 crate 外部不可访问。  
- `pub(super)`：将实体设为在父模块内公共。  
- `pub(in path::to::module)`：将实体设为在指定的模块内公共。

你可以将这些修饰符用于模块、结构体、函数、字段等。例如：

```rust
pub struct Configuration {
    pub(crate) version: u32,
    active: bool,
}
```

`Configuration` 是公共的，但你只能在同一个 crate 内访问其 `version` 字段。  
而 `active` 字段是私有的，只能在同一个模块或它的某个子模块内访问。
