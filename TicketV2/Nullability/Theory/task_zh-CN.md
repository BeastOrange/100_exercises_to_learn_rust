## 可空性

我们对 `assigned` 方法的实现相当简单：对待办和已完成的任务触发恐慌（panic）显然并不理想。  
我们可以使用 **Rust 的 `Option` 类型** 进行更好的处理。

## `Option`

`Option` 是 Rust 中用来表示 **可空值** 的一种类型。  
它是一个枚举类型，定义在 Rust 的标准库中：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

`Option` 表达的是一个值可能存在（`Some(T)`）也可能不存在（`None`）的概念。  
它还强制你 **显式处理这两种情况**。如果你在处理可空值时没有考虑到 `None` 的情况，
编译器会报错。  
相比其他语言中的“隐式”可空性（你可能忘记检查 `null` 从而引发运行时错误），这是一个显著的改进。

## `Option` 的定义

`Option` 的定义使用了一个你可能还未见过的 Rust 构造：**元组型变体**。

### 元组型变体

`Option` 有两个变体：`Some(T)` 和 `None`。  
`Some` 是一个 **元组型变体**：它是一个包含 **未命名字段** 的变体。

元组型变体通常用于存储单个字段，特别是在像 `Option` 这样的“包装器”类型中。

### 元组型结构体

它们不仅适用于枚举——你也可以定义元组型结构体：

```rust
struct Point(i32, i32);
```

然后，你可以使用位置索引访问 `Point` 实例的两个字段：

```rust
let point = Point(3, 4);
let x = point.0;
let y = point.1;
```

### 元组

在我们还没见过元组的情况下，直接提到“元组型”可能显得有些奇怪！  
元组是另一种 Rust 的基础类型。它们将一组固定数量的值组合在一起，这些值可以有不同的类型：

```rust
// 两个值，相同类型
let first: (i32, i32) = (3, 4);
// 三个值，不同类型
let second: (i32, u32, u8) = (-42, 3, 8);
```

其语法很简单：在括号中列出值的类型，用逗号分隔。  
你可以使用点符号和字段索引来访问元组的字段：

```rust
assert_eq!(second.0, -42);
assert_eq!(second.1, 3);
assert_eq!(second.2, 8);
```

当你不愿意专门定义一个结构体类型时，元组是一个方便的方式来将值组合在一起。
